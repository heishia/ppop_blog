# Use Cases

## 사용자 행동 기반 유스케이스

### 1. 키워드 주제 선정

**시나리오**
사용자가 블로그 포스팅을 위한 키워드를 선정합니다.

**주요 흐름**
1. 시스템이 키워드 목록을 제공합니다
2. 사용자가 포스팅 주제로 사용할 키워드를 선택합니다
3. 선택된 키워드가 포스팅 생성에 사용됩니다

**대안 흐름**
- 사용자가 새 키워드를 직접 생성할 수 있습니다
- 기존 키워드를 수정하거나 삭제할 수 있습니다

### 2. 글 검토

**시나리오**
시스템이 자동 생성한 글을 사용자가 검토합니다.

**주요 흐름**
1. 시스템이 키워드를 기반으로 글을 자동 생성합니다
2. 생성된 글 목록을 사용자에게 제공합니다
3. 사용자가 특정 글을 선택하여 내용을 검토합니다
4. 사용자가 필요시 글 내용을 수정합니다
5. 검토 완료 후 다음 단계(이미지 검토)로 진행합니다

**대안 흐름**
- 사용자가 글을 삭제할 수 있습니다
- 사용자가 새 글을 직접 작성할 수 있습니다

### 3. 이미지 검토

**시나리오**
포스팅에 첨부될 이미지를 사용자가 검토합니다.

**주요 흐름**
1. 시스템이 포스팅에 첨부할 이미지 목록을 제공합니다
2. 사용자가 각 이미지를 확인합니다
3. 사용자가 필요시 이미지를 추가하거나 삭제합니다
4. 검토 완료 후 다음 단계(발행 여부 선택)로 진행합니다

**대안 흐름**
- 사용자가 새 이미지를 업로드할 수 있습니다
- 사용자가 이미지 정보를 수정할 수 있습니다

### 4. 글 발행 여부 선택

**시나리오**
검토가 완료된 포스팅의 발행 여부를 사용자가 결정합니다.

**주요 흐름**
1. 사용자가 검토 완료된 포스팅(글 + 이미지)을 확인합니다
2. 사용자가 발행 여부를 선택합니다
   - 발행 선택: 네이버 블로그에 자동 업로드
   - 발행 보류: 임시 저장 또는 삭제
3. 발행 선택 시 시스템이 네이버 블로그에 로그인합니다
4. 시스템이 포스팅을 업로드합니다
5. 업로드 결과를 사용자에게 알립니다

**대안 흐름**
- 로그인 실패 시 사용자에게 알림
- 업로드 실패 시 재시도 옵션 제공
- 발행 보류 시 나중에 다시 발행할 수 있도록 저장

## 유스케이스 다이어그램

```
[사용자] --> [키워드 주제 선정]
[사용자] --> [글 검토]
[사용자] --> [이미지 검토]
[사용자] --> [글 발행 여부 선택]
[글 발행 여부 선택] --> [네이버 블로그 로그인]
[글 발행 여부 선택] --> [포스팅 업로드]
```
